Hexo博客优化之路（一）

发表于 2019-01-31 | 更新于: 2019-02-01 | 分类于 技术

路漫漫其修远兮

经过之前的Hexo基础搭建，我们已经拥有了一个简单的博客，接下来就是要根据自己喜好，将博客进行优化，通过修改主题的配置文件，添加插件等形式，使得博客内容更加丰富。

Hexo配置文件

  • 位于站点根目录下的 _config.yml 文件，可以设置Hexo的各种配置，具体条目可以根据Hexo官网关于 配置 的介绍，很方便、清晰地完成配置。

主题配置文件

  • 在Hexo中有很多非常不错的主题，在这里只介绍了其中一款比较流行的主题：NexT，由于它的开发成员中有很多中国人，所以其对中文有很好的支持，下面就对其主题配置进行简单说明。

  1. 主题风格
    NexT主题在# Schemes下提供了4种不同的风格，分别为Muse、Mist、Pisces、Gemini，都是走的简约风格，对于技术博客而言，是不错的选择。

  2. 菜单设置
    在# Menu Settings下，可以设置博客的菜单，其中各设置参数项说明如下（各项参数可以自定义，根据需要增减）：

参数 说明
home 主页相对地址
about 关于博客
tags 标签页
categories 分类页
archives 目录

  1. 工具栏设置
    在#Sidebar Settings下可以设置工具栏信息，按照其‘Key: permalink || icon’的格式设定即可，其中Key表示工具栏展示的名称，permalink是对应的链接，icon则是搭配的图标。

  2. 站点信息设置
    在#Site Information Settings下，可以设置站点的信息：

参数 说明
favicon 站点图标
keywords 站点关键字
rss 站点RSS功能
footer 站点底部设置

  1. 文章设置
    在#Post Settings下，主要是针对博客文章的设置，其中最主要的是excerpt_description参数，因为你的主页不可能展现每一篇文章的全部内容，所以需要在文章中加入 < !– more –>，这样的话，主页就只会展示每篇博客< !– more –>之前的内容。（#Post Settings下还有其他参数，用法在配置文件中都写的很清楚了，稍微会点英文都能看懂，这里就不再赘述）

  2. 第三方服务插件设置
    在#Third Party Services Settings下设置三方插件参数（如第三方评论插件duoshuo），如果需要引入第三方插件功能，需要打开或添加相应的配置。
